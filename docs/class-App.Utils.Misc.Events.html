Class Events

Provides the ability to register and fire events, usefull if you want to seperate out your core functionality and your additional (such as plugins).

// Registers a new function for the 'user_registered'
// event (inside a plugin file for example).
Events::register('user_registered', function($args){
    $user_name = $args[0]; // Would store 'Bob123', see below.
});

// Fires the 'user_registerd' event from your core code, optionally passing in
// arguments to all the registered functions for this particular event.
Events::fire('user_registered', 'Bob123');

# fire( )

Fire the specified event and pass along the corresponding arguments to the registered callback functions. The format for this functions arguments are 'event_name' [, arg1[, arg2]]. For example, firing the event 'system_started' and the time would look like fire('system_started', time()). Any arguments passed to this function after the event name will be passed to the registered callback functions as an array.
